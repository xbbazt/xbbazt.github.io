搭建一个属于自己VPS探针 MyNodeQuery

· 2024-10-20 ·

MyNodeQurey 简介

关于 MyNodeQuery，这是一款轻量级的服务器和网络性能监控工具，特别用于延迟监控。它支持实时监控虚拟专用服务器（VPS）网络状况，支持对节点进行 IPv4 和 IPv6 的 Ping 测试。MyNodeQuery 提供多种更新和优化功能，例如在最新版本 1.0.3.0 中，性能得到了优化，增加了国家地区旗帜的显示功能，并增加了更多的网络监控细节！

搭建MyNodeQuery

  1. 1c512m10g 即可，区域按你的所在位置购买。如果你是在中国大陆，不是很推荐中国大陆服务器。会造成无法访问或下载速度慢种种原因导致无法正常部署。
  2. 域名一枚并托管到DNS服务商。

准备环境

  1. 系统：最好使用yum或apt包管理器Linux系统易于操作。本教程以 Debian系统为例，centos等请替换教程中 apt 字样代码即可。
  2. 软件：Docker，Docker-compose，curl，Vim，socat，Nginx(本教程以Nginx为例),，acme

安装流程

接下来我们废话不多说，直入主题吧！

安装可能使用到的软件

更新系统源数据及软件

apt update && apt upgrade -y

安装软件

apt install curl vim socat -y

安装Docker

curl -fsSL https://get.docker.com | bash -s docker

测试是否安装成功

root@localhost:~# docker -v
Docker version 27.3.1, build ce12230

使用Docker官方一键安装最新版Docker，无需参考官方文档。简单又省力！

安装Docker-compose

curl -L "https://github.com/docker/compose/releases/download/v2.29.6/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

chmod +x /usr/local/bin/docker-compose

测试是否安装成功

root@localhost:/usr/local/bin# docker-compose version
Docker Compose version v2.29.6

创建目录

cd /home
mkdir web
cd web
mkdir mynodequery
mkdir mysql
mkdir certs
touch nginx.conf
touch docker-compose.yml

docker-compose配置文件的编写

services:
  nginx:
    image: nginx
    container_name: nginx
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - /home/web/nginx.conf:/etc/nginx/nginx.conf
      - /home/web/certs:/etc/nginx/certs

  mysql:
    image: mysql
    container_name: mysql # 主机名
    restart: always
    volumes:
      - /home/web/mysql:/var/lib/mysql
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword # root密码
      MYSQL_DATABASE: mynodequery # 数据库名
      MYSQL_USER: xbbazt # 用户名
      MYSQL_PASSWORD: xbbazt # 密码

  mynodequery:
    image: jaydenlee2019/mynodequery:latest
    container_name: mynodequery
    restart: always
    ports:
      - "5000:5000"
    volumes:
      - /home/web/mynodequery/appsettings.json:/app/appsettings.json
    links:
      - mysql

创建mynodequery配置文件

cat > /home/web/mynodequery/appsettings.json <<EOF
{
    "Logging": {
        "LogLevel": {
            "Default": "Information",
            "Microsoft": "Warning",
            "Microsoft.Hosting.Lifetime": "Information"
        }
    },
    "MySql": {
        "ConnectionString": ""
    },
    "AllowedHosts": "*",
    "Installed": "false",
    "ReadNodeIpHeaderKey": "X-Real-IP"
}
EOF

申请证书

  1. 下载acme
curl https://get.acme.sh | sh
  1. 输入你的邮箱
~/.acme.sh/acme.sh --register-account -m xxxx@gmail.com
  1. 申请证书
~/.acme.sh/acme.sh --issue -d 域名 --standalone
  1. 下载证书
~/.acme.sh/acme.sh --installcert -d 域名 --key-file /home/web/certs/key.pem --fullchain-file /home/web/certs/cert.pem

nginx配置

worker_processes  auto;

error_log  /var/log/nginx/error.log warn;
pid        /var/run/nginx.pid;

events {
    worker_connections  1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    access_log  /var/log/nginx/access.log  main;

    sendfile        on;
    keepalive_timeout  65;

    upstream backend {
        server 172.17.0.1:5000;
    }

    server {
        listen       80;
        server_name  域名; # 或者替换为你的域名

        # 301 重定向 HTTP 到 HTTPS
        return 301 https://$host$request_uri;
    }

    server {
        listen       443 ssl;  # 监听 HTTPS 端口
        server_name  域名; 

        ssl_certificate      /etc/nginx/certs/cert.pem;  # 证书文件路径
        ssl_certificate_key  /etc/nginx/certs/key.pem;    # 私钥文件路径

        ssl_protocols       TLSv1.2 TLSv1.3;  # 使用的 TLS 版本
        ssl_ciphers         HIGH:!aNULL:!MD5; # 支持的加密套件

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

启动你的Docker容器

docker-compose up -d

访问

访问刚才配置在nginx配置域名，就可得到如下页面！

  • 主机名：mysql
  • 端口：3306
  • 数据库名：mynodequery
  • 用户ID：xbbazt
  • 密码：xbbazt

站点设置

其他不用修改，只需要为你的探针设置管理员密码即可。

ping 设置

  • IP版本：IPv4/IPv6
  • 协议：ICMP=ping(无需端口)/TCP(需要请求端口)
  • 节点名称：取一个名字
  • 节点地址：域名/IP

部署完毕

只要按照每一步操作几乎100%都可以部署出来，要换作以前没有容器化确实探针部署不是一件容易的事情！
